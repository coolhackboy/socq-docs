# LinkedIn 职位

```json
{
  "urls": ["https://www.linkedin.com/jobs/view/3986111804"]
}
```

{/* mcp-usage:start */}
## 与 MCP 一起使用

| 接口 | 值 |
| --- | --- |
| REST | `POST /v1/linkedin/jobs` |
| Compact MCP | `socq_execute` 与 `endpoint: "linkedin/jobs"` |
| Typed MCP | `socq_linkedin_jobs` |
| 精确工具 URL | `https://api.socq.ai/mcp?tools=linkedin_jobs` |
| CLI | `socq linkedin jobs` |

使用上面的 JSON 请求正文作为 `socq_execute` 的嵌套 `input` 对象，或将相同的字段直接传递给Typed 工具。Typed 工具还接受 `_wait_seconds`、`_result_limit` 和 `_idempotency_key`。

<Info>
  成功的采集将返回 `task_id`。继续 `socq_get_task` 直到任务成功，然后从 `results.items` 读取规范化记录。
</Info>

有关完整的提交和轮询流程，请参阅[MCP 快速入门](/zh/integrations/mcp)。
{/* mcp-usage:end */}

每个 URL 必须标识一个职位详细信息页面。结果使用`resource="jobs"`和`type="job"`；职位、描述、公司、地点、就业类型和申请详细信息均已标准化。
