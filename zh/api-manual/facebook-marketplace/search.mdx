# Marketplace 搜索

搜索地理坐标周围的公开列表。需要时使用[Marketplace 位置搜索](/zh/api-manual/facebook-marketplace/location-search)查找坐标。

## 请求示例

```json
{
  "query": "bike",
  "latitude": 30.2677,
  "longitude": -97.7475,
  "results_limit": 100,
  "radius_km": 65,
  "min_price": 100,
  "max_price": 500,
  "sort_by": "creation_time_descend",
  "delivery_method": "local_pickup",
  "condition": "used_good",
  "date_listed": "last_7_days",
  "availability": "available"
}
```

{/* mcp-usage:start */}
## 与 MCP 一起使用

| 接口 | 值 |
| --- | --- |
| REST | `POST /v1/facebook-marketplace/search` |
| Compact MCP | `socq_execute` 与 `endpoint: "facebook-marketplace/search"` |
| Typed MCP | `socq_facebook_marketplace_search` |
| 精确工具 URL | `https://api.socq.ai/mcp?tools=facebook_marketplace_search` |
| CLI | `socq facebook-marketplace search` |

使用上面的 JSON 请求正文作为 `socq_execute` 的嵌套 `input` 对象，或将相同的字段直接传递给Typed 工具。Typed 工具还接受 `_wait_seconds`、`_result_limit` 和 `_idempotency_key`。

<Info>
  成功的采集将返回 `task_id`。继续 `socq_get_task` 直到任务成功，然后从 `results.items` 读取规范化记录。
</Info>

有关完整的提交和轮询流程，请参阅[MCP 快速入门](/zh/integrations/mcp)。
{/* mcp-usage:end */}

SocQ 遵循结果分页，直到到达 `results_limit` 或没有其他页面可用。重复的列表 ID 会保存一次，因此最终的 `result_count` 可能会低于请求的限制。
