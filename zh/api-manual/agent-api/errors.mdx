# 错误

代理 API 错误使用一致的响应信封。

```json
{
  "code": 400,
  "error": {
    "message": "invalid_field_type:urls, expected array",
    "type": "bad_request"
  }
}
```

## 常用代码

| HTTP 代码 |意义|
| --- | --- |
| `400` |无效的请求正文或不受支持的字段 |
| `401` | API 密钥缺失或无效 |
| `402` |积分不足|
| `403` |任务不属于经过身份验证的账户 |
| `404` |未找到端点功能或任务 |
| `429` |超出利率限制或积分 |
| `500` |任务提交失败 |

## 处理限制错误

`402` 响应意味着该账户没有足够的积分来满足该请求。

`429` 响应可能指示 API 密钥的每分钟请求限制或每小时或每日积分限额。当 `Retry-After` 存在时，请尊重它，并在重试之前等待适用的窗口。减少 `results_limit` 可以降低所需的积分，但不会绕过请求率限制。

不要盲目重试。如果在客户端或网络超时之前提交可能已成功，请查询现有任务 ID 或使用幂等键重试，以避免重复收费。
