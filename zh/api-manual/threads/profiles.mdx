# Threads 简介

```json
{
  "urls": ["https://www.threads.com/@example"]
}
```

{/* mcp-usage:start */}
## 与 MCP 一起使用

| 接口 | 值 |
| --- | --- |
| REST | `POST /v1/threads/profiles` |
| Compact MCP | `socq_execute` 与 `endpoint: "threads/profiles"` |
| Typed MCP | `socq_threads_profiles` |
| 精确工具 URL | `https://api.socq.ai/mcp?tools=threads_profiles` |
| CLI | `socq threads profiles` |

使用上面的 JSON 请求正文作为 `socq_execute` 的嵌套 `input` 对象，或将相同的字段直接传递给Typed 工具。Typed 工具还接受 `_wait_seconds`、`_result_limit` 和 `_idempotency_key`。

<Info>
  成功的采集将返回 `task_id`。继续 `socq_get_task` 直到任务成功，然后从 `results.items` 读取规范化记录。
</Info>

有关完整的提交和轮询流程，请参阅[MCP 快速入门](/zh/integrations/mcp)。
{/* mcp-usage:end */}

结果使用 `resource="profiles"` 和 `type="profile"`，包含个人资料名称、用户名、简介、网站和关注者指标。
