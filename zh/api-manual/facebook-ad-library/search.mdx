# 广告搜索

按关键字、状态、国家/地区、媒体类型和日期范围搜索公共广告。

## 请求示例

```json
{
  "query": "running shoes",
  "results_limit": 100,
  "country": "US",
  "status": "ACTIVE",
  "media_type": "VIDEO",
  "sort_by": "total_impressions"
}
```

{/* mcp-usage:start */}
## 与 MCP 一起使用

| 接口 | 值 |
| --- | --- |
| REST | `POST /v1/facebook-ad-library/search` |
| Compact MCP | `socq_execute` 与 `endpoint: "facebook-ad-library/search"` |
| Typed MCP | `socq_facebook_ad_library_search` |
| 精确工具 URL | `https://api.socq.ai/mcp?tools=facebook_ad_library_search` |
| CLI | `socq facebook-ad-library search` |

使用上面的 JSON 请求正文作为 `socq_execute` 的嵌套 `input` 对象，或将相同的字段直接传递给Typed 工具。Typed 工具还接受 `_wait_seconds`、`_result_limit` 和 `_idempotency_key`。

<Info>
  成功的采集将返回 `task_id`。继续 `socq_get_task` 直到任务成功，然后从 `results.items` 读取规范化记录。
</Info>

有关完整的提交和轮询流程，请参阅[MCP 快速入门](/zh/integrations/mcp)。
{/* mcp-usage:end */}

SocQ 遵循可用结果页面，直到到达 `results_limit`、没有下一页可用或任务到达其分页安全边界。重复的广告 ID 会保存一次。
