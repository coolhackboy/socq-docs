# Facebook 公共主页

采集公共 Facebook 页面/个人资料元数据、公共 URL、个人资料字段和相关公共指标。

## 端点

```http
POST https://api.socq.ai/v1/facebook/pages
```

## 请求正文

```json
{
  "urls": ["https://www.facebook.com/facebook"]
}
```

{/* mcp-usage:start */}
## 与 MCP 一起使用

| 接口 | 值 |
| --- | --- |
| REST | `POST /v1/facebook/pages` |
| Compact MCP | `socq_execute` 与 `endpoint: "facebook/pages"` |
| Typed MCP | `socq_facebook_pages` |
| 精确工具 URL | `https://api.socq.ai/mcp?tools=facebook_pages` |
| CLI | `socq facebook pages` |

使用上面的 JSON 请求正文作为 `socq_execute` 的嵌套 `input` 对象，或将相同的字段直接传递给Typed 工具。Typed 工具还接受 `_wait_seconds`、`_result_limit` 和 `_idempotency_key`。

<Info>
  成功的采集将返回 `task_id`。继续 `socq_get_task` 直到任务成功，然后从 `results.items` 读取规范化记录。
</Info>

有关完整的提交和轮询流程，请参阅[MCP 快速入门](/zh/integrations/mcp)。
{/* mcp-usage:end */}

## 结果项

任务成功后，调用`GET /v1/tasks/{task_id}`读取`results`下的分页标准记录。

```json
{
  "id": "100064860875397",
  "platform": "facebook",
  "resource": "pages",
  "type": "page",
  "name": "Facebook",
  "url": "https://www.facebook.com/facebook/",
  "category": "Internet company",
  "about": "Public page description",
  "avatar_url": "https://example.com/avatar.jpg",
  "cover_url": "https://example.com/cover.jpg",
  "metrics": {
    "followers_count": 154000000,
    "likes_count": 154831857,
    "reviews_count": null,
    "rating": null
  },
  "contact": {
    "website": "https://example.com",
    "email": null,
    "phone": null,
    "address": null
  },
  "created_at": "2007-11-07T00:00:00",
  "collected_at": "2026-07-08T08:59:45",
```

您可以在同一字段中传递多个页面 URL：

```json
{
  "urls": [
    "https://www.facebook.com/facebook",
    "https://www.facebook.com/Meta"
  ]
}
```
