# Pinterest Pin

```json
{
  "urls": ["https://www.pinterest.com/pin/123456789/"]
}
```

{/* mcp-usage:start */}
## 与 MCP 一起使用

| 接口 | 值 |
| --- | --- |
| REST | `POST /v1/pinterest/pins` |
| Compact MCP | `socq_execute` 与 `endpoint: "pinterest/pins"` |
| Typed MCP | `socq_pinterest_pins` |
| 精确工具 URL | `https://api.socq.ai/mcp?tools=pinterest_pins` |
| CLI | `socq pinterest pins` |

使用上面的 JSON 请求正文作为 `socq_execute` 的嵌套 `input` 对象，或将相同的字段直接传递给Typed 工具。Typed 工具还接受 `_wait_seconds`、`_result_limit` 和 `_idempotency_key`。

<Info>
  成功的采集将返回 `task_id`。继续 `socq_get_task` 直到任务成功，然后从 `results.items` 读取规范化记录。
</Info>

有关完整的提交和轮询流程，请参阅[MCP 快速入门](/zh/integrations/mcp)。
{/* mcp-usage:end */}

结果使用 `resource="pins"` 和 `type="pin"`。 Pin 内容在 `text` 中返回，其中包含标准结果结构中的作者、指标、媒体、类别和主题标签。
