# YouTube 搜索

使用稳定的、独立于数据提供方的过滤器搜索公共视频。

```json
{
  "query": "web scraping tutorials",
  "results_limit": 50,
  "sort_by": "relevance",
  "published_within": "month",
  "duration": "medium",
  "has_captions": true
}
```

{/* mcp-usage:start */}
## 与 MCP 一起使用

| 接口 | 值 |
| --- | --- |
| REST | `POST /v1/youtube/search` |
| Compact MCP | `socq_execute` 与 `endpoint: "youtube/search"` |
| Typed MCP | `socq_youtube_search` |
| 精确工具 URL | `https://api.socq.ai/mcp?tools=youtube_search` |
| CLI | `socq youtube search` |

使用上面的 JSON 请求正文作为 `socq_execute` 的嵌套 `input` 对象，或将相同的字段直接传递给Typed 工具。Typed 工具还接受 `_wait_seconds`、`_result_limit` 和 `_idempotency_key`。

<Info>
  成功的采集将返回 `task_id`。继续 `socq_get_task` 直到任务成功，然后从 `results.items` 读取规范化记录。
</Info>

有关完整的提交和轮询流程，请参阅[MCP 快速入门](/zh/integrations/mcp)。
{/* mcp-usage:end */}

|领域|接受的值|
| --- | --- |
| `sort_by` | `relevance`、`date`、`rating`、`view_count` |
| `published_within` | `hour`、`day`、`week`、`month`、`year` |
| `duration` | `short`、`medium`、`long` |

`results_limit` 默认为 100。

结果使用 [YouTube 视频](/zh/api-manual/youtube/videos) 中记录的标准化视频结构，其中 `resource` 设置为 `search`。
