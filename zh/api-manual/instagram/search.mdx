# Instagram 搜索

搜索公共 Instagram 个人资料以获取用户名和创建者发现工作流程。

## 端点

```http
POST https://api.socq.ai/v1/instagram/search
```

## 请求正文

```json
{
  "query": "instagram",
  "results_limit": 20
}
```

{/* mcp-usage:start */}
## 与 MCP 一起使用

| 接口 | 值 |
| --- | --- |
| REST | `POST /v1/instagram/search` |
| Compact MCP | `socq_execute` 与 `endpoint: "instagram/search"` |
| Typed MCP | `socq_instagram_search` |
| 精确工具 URL | `https://api.socq.ai/mcp?tools=instagram_search` |
| CLI | `socq instagram search` |

使用上面的 JSON 请求正文作为 `socq_execute` 的嵌套 `input` 对象，或将相同的字段直接传递给Typed 工具。Typed 工具还接受 `_wait_seconds`、`_result_limit` 和 `_idempotency_key`。

<Info>
  成功的采集将返回 `task_id`。继续 `socq_get_task` 直到任务成功，然后从 `results.items` 读取规范化记录。
</Info>

有关完整的提交和轮询流程，请参阅[MCP 快速入门](/zh/integrations/mcp)。
{/* mcp-usage:end */}

## 结果项

任务成功后，调用`GET /v1/tasks/{task_id}`读取`results`下的分页档案记录。

```json
{
  "id": "25025320",
  "platform": "instagram",
  "resource": "search",
  "type": "profile",
  "username": "instagram",
  "name": "Instagram",
  "url": "https://www.instagram.com/instagram",
  "avatar_url": "https://example.com/avatar.jpg",
  "bio": "Discover what's next on Instagram",
  "is_verified": true,
  "is_private": false,
  "metrics": {
    "followers_count": 685829413,
    "following_count": 251,
    "posts_count": 8000
  },
  "created_at": null,
  "collected_at": "2026-07-08T08:59:45",
  "extra": {}
}
```

`query` 作为公共个人资料/用户关键字进行搜索。 `results_limit` 控制个人资料搜索结果的最大数量。
