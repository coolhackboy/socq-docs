# Instagram Followers Count

Collect public follower and following counts for Instagram profiles.

## Endpoint

```http
POST https://api.socq.ai/v1/instagram/followers-count
```

## Request Body

```json
{
  "usernames": ["instagram"]
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/instagram/followers-count` |
| Compact MCP | `socq_execute` with `endpoint: "instagram/followers-count"` |
| Typed MCP | `socq_instagram_followers_count` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=instagram_followers_count` |
| CLI | `socq instagram followers-count` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

## Result Item

After the task succeeds, call `GET /v1/tasks/{task_id}` to read paginated profile metric records under `results`.

```json
{
  "id": "25025320",
  "platform": "instagram",
  "resource": "followers-count",
  "type": "profile_metrics",
  "username": "instagram",
  "name": "Instagram",
  "url": "https://www.instagram.com/instagram",
  "avatar_url": "https://example.com/avatar.jpg",
  "is_verified": true,
  "is_private": false,
  "metrics": {
    "followers_count": 685829413,
    "following_count": 251,
    "posts_count": null
  },
  "created_at": null,
  "collected_at": "2026-07-08T08:59:45",
  "extra": {
    "snapshot_time": "2026-07-08 - 02:02"
  }
}
```

You can submit multiple usernames in the same field:

```json
{
  "usernames": ["instagram", "creators"]
}
```
