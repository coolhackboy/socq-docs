# X Search

Search public posts using a keyword or search expression.

```json
{
  "query": "AI lang:en",
  "results_limit": 100,
  "sort_by": "latest"
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/x/search` |
| Compact MCP | `socq_execute` with `endpoint: "x/search"` |
| Typed MCP | `socq_x_search` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=x_search` |
| CLI | `socq x search` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

| Field | Accepted values |
| --- | --- |
| `sort_by` | `latest`, `top` |

Results use the normalized structure documented in [X Posts](/api-manual/x/posts), with `resource` set to `search`. The submitted expression is returned under `extra.search_query`.
