# TikTok User Showcase

Collect products displayed in one public TikTok user's Shop showcase.

## Request Example

```json
{
  "username": "mrtiktokreviews",
  "results_limit": 100,
  "region": "US"
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/tiktok-shop/user-showcase` |
| Compact MCP | `socq_execute` with `endpoint: "tiktok-shop/user-showcase"` |
| Typed MCP | `socq_tiktok_shop_user_showcase` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=tiktok_shop_user_showcase` |
| CLI | `socq tiktok-shop user-showcase` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

One task accepts one username. The `@` prefix is optional.
