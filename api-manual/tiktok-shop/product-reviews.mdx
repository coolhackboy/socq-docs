# TikTok Shop Product Reviews

Collect public review text, ratings, reviewer information, images, and purchase status.

## Request Example

```json
{
  "url": "https://www.tiktok.com/shop/pdp/example-product/1729587769570529799",
  "results_limit": 100,
  "region": "US"
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/tiktok-shop/product-reviews` |
| Compact MCP | `socq_execute` with `endpoint: "tiktok-shop/product-reviews"` |
| Typed MCP | `socq_tiktok_shop_product_reviews` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=tiktok_shop_product_reviews` |
| CLI | `socq tiktok-shop product-reviews` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

One task accepts one product URL. Duplicate review IDs are saved once and all saved reviews participate in billing.
