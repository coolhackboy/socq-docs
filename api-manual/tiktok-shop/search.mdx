# TikTok Shop Search

Search public products by keyword and market region.

## Request Example

```json
{
  "query": "running shoes",
  "results_limit": 100,
  "region": "US"
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/tiktok-shop/search` |
| Compact MCP | `socq_execute` with `endpoint: "tiktok-shop/search"` |
| Typed MCP | `socq_tiktok_shop_search` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=tiktok_shop_search` |
| CLI | `socq tiktok-shop search` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

SocQ requests result pages until `results_limit` is reached or no additional products are available. Duplicate product IDs are saved once.
