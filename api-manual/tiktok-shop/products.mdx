# TikTok Shop Products

Collect products from one store URL, sorted by best sellers or newest releases.

## Request Example

```json
{
  "url": "https://www.tiktok.com/shop/store/goli-nutrition/7495794203056835079",
  "results_limit": 100,
  "sort_by": "top",
  "region": "US"
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/tiktok-shop/products` |
| Compact MCP | `socq_execute` with `endpoint: "tiktok-shop/products"` |
| Typed MCP | `socq_tiktok_shop_products` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=tiktok_shop_products` |
| CLI | `socq tiktok-shop products` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

One task accepts one store URL. The final result can contain fewer records than `results_limit` when no additional page is available.
