# Threads Posts

```json
{
  "urls": ["https://www.threads.com/@example/post/POST_ID"]
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/threads/posts` |
| Compact MCP | `socq_execute` with `endpoint: "threads/posts"` |
| Typed MCP | `socq_threads_posts` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=threads_posts` |
| CLI | `socq threads posts` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

Results use `resource="posts"` and `type="post"`. Post content is returned in `text` with author, engagement metrics, media, and publication time.
