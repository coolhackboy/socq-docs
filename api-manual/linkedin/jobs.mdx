# LinkedIn Jobs

```json
{
  "urls": ["https://www.linkedin.com/jobs/view/3986111804"]
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/linkedin/jobs` |
| Compact MCP | `socq_execute` with `endpoint: "linkedin/jobs"` |
| Typed MCP | `socq_linkedin_jobs` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=linkedin_jobs` |
| CLI | `socq linkedin jobs` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

Each URL must identify a job details page. Results use `resource="jobs"` and `type="job"`; title, description, company, location, employment type, and application details are normalized.
