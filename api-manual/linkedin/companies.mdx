# LinkedIn Companies

```json
{
  "urls": ["https://www.linkedin.com/company/example"]
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/linkedin/companies` |
| Compact MCP | `socq_execute` with `endpoint: "linkedin/companies"` |
| Typed MCP | `socq_linkedin_companies` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=linkedin_companies` |
| CLI | `socq linkedin companies` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

Results use `platform="linkedin"`, `resource="companies"`, and `type="company"`. Industry, company size, headquarters, locations, and specialties are returned under `extra`.
