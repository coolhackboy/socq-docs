# YouTube Shorts

Collect Shorts from one or more public channels.

```json
{
  "urls": ["https://www.youtube.com/@example"],
  "results_limit": 100
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/youtube/shorts` |
| Compact MCP | `socq_execute` with `endpoint: "youtube/shorts"` |
| Typed MCP | `socq_youtube_shorts` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=youtube_shorts` |
| CLI | `socq youtube shorts` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

`results_limit` defaults to 100 and applies to each channel URL.

Results use the normalized video structure documented in [YouTube Videos](/api-manual/youtube/videos), with `resource` set to `shorts` and `type` set to `short`.
