# Pinterest Search

```json
{
  "query": "home office ideas",
  "results_limit": 100
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/pinterest/search` |
| Compact MCP | `socq_execute` with `endpoint: "pinterest/search"` |
| Typed MCP | `socq_pinterest_search` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=pinterest_search` |
| CLI | `socq pinterest search` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

`results_limit` defaults to 100 and accepts values from 1 through 2,000. Results use `resource="search"` and `type="pin"`.
