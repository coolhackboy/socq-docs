# Pinterest Profiles

```json
{
  "urls": ["https://www.pinterest.com/example/"]
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/pinterest/profiles` |
| Compact MCP | `socq_execute` with `endpoint: "pinterest/profiles"` |
| Typed MCP | `socq_pinterest_profiles` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=pinterest_profiles` |
| CLI | `socq pinterest profiles` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

Results use `resource="profiles"` and `type="profile"`, with profile name, username, image, biography, website, and account metrics.
