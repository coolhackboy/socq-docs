# Ad Library Search

Search public ads by keyword, status, country, media type, and date range.

## Request Example

```json
{
  "query": "running shoes",
  "results_limit": 100,
  "country": "US",
  "status": "ACTIVE",
  "media_type": "VIDEO",
  "sort_by": "total_impressions"
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/facebook-ad-library/search` |
| Compact MCP | `socq_execute` with `endpoint: "facebook-ad-library/search"` |
| Typed MCP | `socq_facebook_ad_library_search` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=facebook_ad_library_search` |
| CLI | `socq facebook-ad-library search` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

SocQ follows available result pages until `results_limit` is reached, no next page is available, or the task reaches its pagination safety boundary. Duplicate ad IDs are saved once.
