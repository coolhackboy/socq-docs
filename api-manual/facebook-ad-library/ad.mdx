# Ad Details

Collect creative, page, media, delivery status, and timing data for one public ad URL.

## Request Example

```json
{
  "url": "https://www.facebook.com/ads/library/?id=702369045530963"
}
```

{/* mcp-usage:start */}
## Use with MCP

| Interface | Value |
| --- | --- |
| REST | `POST /v1/facebook-ad-library/ad` |
| Compact MCP | `socq_execute` with `endpoint: "facebook-ad-library/ad"` |
| Typed MCP | `socq_facebook_ad_library_ad` |
| Exact-tool URL | `https://api.socq.ai/mcp?tools=facebook_ad_library_ad` |
| CLI | `socq facebook-ad-library ad` |

Use the JSON request body above as the nested `input` object for `socq_execute`, or pass the same fields directly to the typed tool. Typed tools also accept `_wait_seconds`, `_result_limit`, and `_idempotency_key`.

<Info>
  A successful collection returns a `task_id`. Continue with `socq_get_task` until the task succeeds, then read normalized records from `results.items`.
</Info>

See [MCP quickstart](/integrations/mcp) for the complete submission and polling flow.
{/* mcp-usage:end */}

One task accepts one ad URL and produces at most one normalized ad record.
